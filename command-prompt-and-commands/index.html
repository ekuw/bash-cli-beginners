The command prompt and commands

The command prompt and commands explained

A terminal provides a command prompt, which represents the input field for users to enter commands. The prompt also provides useful orientation information to the user.

The command prompt

The command prompt denotes the location of expected user input in the command line terminal. As noted in earlier sections, users cannot change the position of this using a mouse. A typical command prompt might look something like this: 

bob@IT1:~$
Breaking this down:
Username bob This is the username of the computing account holder.
Computer name IT1 This is the name of the computer, or the hostname.
Current location ~ This is the users current location in the file system, i.e. the path of the current directory. Recall that ~ is a special character? It denotes the home directory, which is usually the location in which users will find themselves by default when launching a terminal session.
$ $ This signifies that the terminal is ready for user input. There should be a flashing cursor to the right of the dollar sign. Code snippets provided online usually include a leading $. In most cases this should not be interpreted as part of the command, but rather a representation of the command prompt itself.

Commands, options and arguments

Most of the time a command issued on the command line will have the following syntax: 

command (options) arguments
for example: 

grep -R "Curriculum Vitae" Documents
Here the command is grep, a super-useful command that will search plain text data files for lines that match a given regular expression. In this case that regular expression is "Curriculum Vitae". The -R is an option or flag, and these are usually specified with a leading hyphen. The available options depend on the command. In this case the -R indicates that a recursive search has been requested (usually grep takes single files as input, but the recursive option allows whole directories to be passed to grep). In this example the search is being performed over all text-based files in the "Documents" directory. Both "Documents" and "Curriculum Vitae" are arguments that have been passed to the grep command.
