Why use the command line?

What are some of the benefits that learning to interact with software through a command line interface can bring?

Command line interfaces, or shells, provide a terminal or console into which a user can issue instructions and provide data to a computer operating system interactively. The level of precision afforded by this method of user input means that tasks can be made reproducible, scalable and can be documented without the ambiguity sometimes introduced when navigating Graphical User Interfaces (GUIs).

Why use the command line?

Using the command line interface gives users the ability to deliver text-based commands to an operating system. Because these commands are spelled out explicitly, they are clear and unambiguous. This means that tasks can be well defined, and subsequently reproduced consistently. Repeatedly. Users therefore have greater control, and can:

  • Give clear and precise instructions to operating systems.
  • Repeat tasks and reproduce the result exactly by issuing the same command that was used to perform the original task.
  • Clearly document task methodologies by recording the necessary commands, and possibly integrating them into a bash script.
  • Automate tasks and operations. The same command(s) can be issued to multiple files, multiple times.

    • Beyond the benefits listed above, learning how to interact with computers via the command line introduces users to the logic and language of computer programming. This is hugely important for not only developers, but also anyone who does any kind of analytical work using code. A basic understanding will help uncover the true power available from your computer system, and will be vital to efficiently debug issues that come up when performing analytical tasks. The command line makes data science reproducible and scalable, and ultimately makes the user more efficient.

    Use of software packages

    Whether you're a developer, data scientist or analyst, you're likely to make use of mainstream software packages and tools in the course of your daily work.

    Git, for example, is a modern version control system that has been broadly adopted as the go-to for version control. Though GUIs for Git do exist, Git was originally designed for command line use, and the full functionality of Git can be only be exploited via the command line. Generally only partial functionality can be offered through GUIs. To understand Git properly, having a basic working knowledge of the command line will be extremely helpful.

    When writing a computer program, or analysis script, Integrated Development Enviroments (IDEs) enable programmers to write code with the aid of syntax highlighting and autocomplete, while providing (at a minimum) build and debugging tools in the same application.
