Things to remember

Some tips, tricks, and things to keep in mind when using the command line interface

The command line interface can seem intimidating for users who are used to navigating their operating systems using Graphical User Interfaces (GUIs). Some methods and conventions may appear counter-intuitive. In this section we note some of the features of the command line that might not be obvious to beginners, and provide a glossary of some useful tips and shortcuts.

Navigation

When navigating a computer file system using a GUI, users typically click through a graphical representation of the file system to select folders and open files using the mouse. This could be using e.g. Windows Explorer on a Windows machine or Finder on OSX. To use the command line will mean using the mouse less and the keyboard more. You cannot position your cursor on the command line terminal by moving your mouse and clicking at a given location. The left and right arrows can be used to move the cursor horizontally along a line of text, and the up and down arrows can be used to scroll through your command history.

Tab completion is also a thing on the command line, so if you start to type a command (or the path to a folder, a file name, ...) and hit the tab key it will autocomplete (or list autocompletion options). Use of this feature will save time and typos, so get comfortable using it.

Here are some handy navigation tricks to remember:

Function

Command
Jump to the beginning of a line Ctrl-A
Jump to the end of a line Ctrl-E
Delete character under the cursor Ctrl-D or hit delete
Clear the terminal screen Ctrl-L or type clear
Kill the current process running in the terminal Ctrl-C
Suspend/pause the current process running in the terminal Ctrl-Z
Close the bash terminal Ctrl-D or type exit
Cut the part of the current line after the cursor Ctrl-K
Paste the last thing you cut from the clipboard (here the y means "yank") Ctrl-Y

Special characters and whitespace

Some characters have special meanings when used on the command line, and it is important to be aware of these special characters. Although a list containing the meanings of all special characters is out-of-scope for this introductory material, here are some examples of special characters: 
*~#`$&() <>{};'"\|/
Note that the above list includes the whitespace. This is because commands and arguments that are passed to the command line are separated by a whitespace. The shell splits the user input on whitespaces into individual pieces. From these pieces the shell determines what instruction to issue to the operating system. The whitespace can therefore be thought of as the command line separator character, which makes the inclusion of whitespaces in file names and directory paths problematic. The presence of whitespace in these areas often causes issues for new users learning to use the command line. To avoid hitting such issues try to get into the habit of using underscores in place of whitespace in file, directory and variable names. Note that the presence of a whitespace in file or directory name will also cause problems for tab-completion usage (the autocompletion will halt at the whitespace, which will then need to be escaped, as discussed below).

With the whitespace caveat addressed, it is helpful at this stage to simply be aware that these characters have special meanings. This means that if you want to use these characters within a string, i.e. without using their special meanings, then you need to treat these characters... well, specially.

One way of using these characters literally is to use the backslash-escape method. To backslash-escape a special character a backslash needs to immediately precede the character in question. For example, to backslash-escape the wildcard character *, one would type \*.

Another method is to quote the character. This means doing exactly as it sounds, and surrounding the special character with quotation marks. For the example above, this would mean typing "*". Similarly, when faced with a folder called My Files, you would need to escape the special meaning of the whitespace by quoting it, typing "My Files".

Words of caution

The command line represents a way for you to deliver commands to your operating system. This means that any operation that you perform using the command line is a real operation. That is, if you delete that folder using the command line, that is just the same as if you right-click in Windows Explorer and delete that folder, but there is no recycle bin and no undo button. So take care with your commands, make sure you understand what you are doing, as some can cause permanent data loss.
