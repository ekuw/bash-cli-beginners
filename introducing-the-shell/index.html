Introducing the shell

The kernel, the shell and the command line terminal

User-issued commands are entered into the terminal for the shell to interpret and translate so that they can be communicated to the operating system.

The kernel and the shell
Kernel The kernel is at the core of a computer operating system, controlling memory allocation, the running of processes and the system hardware. It is the layer of the operating system that connects the system hardware to the computer software.
Shell The shell is the layer between the user and the kernel. It provides an interface for the the user to issue commands to the operating system.
Bash Bash stands for Bourne again shell, and is a unix shell and command language. Bash is a POSIX (Portable Operating System Interface) compliant shell, meaning that it conforms to standards set out to maintain compatibility between operating systems.
Terminal The terminal can be thought of as the graphical representation of the shell. It is the application with which users can issue text-based commands that the shell can then interpret and translate before passing them to the kernel.
